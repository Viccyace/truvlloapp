# Truvllo PWA Setup Guide

## Install dependencies

```bash
npm install vite-plugin-pwa workbox-window
npm install -D @vite-pwa/assets-generator
```

---

## Generate icons

Add this script to `package.json`:

```json
{
  "scripts": {
    "generate-pwa-assets": "pwa-assets-generator --preset minimal public/logo.svg"
  }
}
```

Then run:

```bash
npm run generate-pwa-assets
```

This generates all required icon sizes in `public/icons/` from a single SVG source.

Alternatively, use https://realfavicongenerator.net with your logo and place the output in `public/`.

Required files in `public/`:

## How the preloader works

`Preloader.jsx` uses `sessionStorage` to show the splash screen only once per browser session:

```
First visit:
  sessionStorage has no key
  → Show preloader (2.1s animated bar + logo)
  → Set sessionStorage key
  → Fade out → app renders

Refresh / subsequent navigation:
  sessionStorage key exists
  → Skip preloader entirely
  → App renders immediately

Tab close:
  sessionStorage is cleared automatically by the browser
  → Next visit shows preloader again (correct behaviour)
```

## How the install prompt works

**Android / Chrome:**
- Browser fires `beforeinstallprompt` when install criteria are met
- We intercept it, prevent default, and show our branded banner
- "Install" button triggers the native Chrome install dialog
- "Not now" dismisses for this session only
- Dismissed via native dialog stores permanently in `localStorage`

**iOS Safari:**
- iOS doesn't support `beforeinstallprompt`
- We detect iOS and show manual step-by-step instructions
- Dismissed permanently via `localStorage`

**Already installed:**
- `window.matchMedia("(display-mode: standalone)").matches` returns true
- Prompt never shows

## PWA install criteria (Chrome)

Chrome shows the install prompt when ALL of these are true:
- ✅ Served over HTTPS
- ✅ Has a valid web app manifest (auto-generated by vite-plugin-pwa)
- ✅ Has a registered service worker with a fetch handler
- ✅ Icons of 192x192 and 512x512 are present
- ✅ `start_url` is within scope
- ✅ User has engaged with the page (30+ seconds)

## PWA checklist

Run `npx lighthouse https://truvllo.vercel.app --only-categories=pwa` after deploy.

- [ ] Manifest is valid and linked in `<head>`
- [ ] Service worker registered and active
- [ ] 192x192 and 512x512 maskable icons present
- [ ] `theme-color` meta tag set
- [ ] `start_url` returns 200 when offline
- [ ] App works offline (shows cached dashboard or offline fallback)
- [ ] No mixed content (all assets over HTTPS)
- [ ] Viewport meta tag present
- [ ] `apple-mobile-web-app-capable` set

## Offline behaviour

With the Workbox config in `vite.config.js`:

| Resource | Strategy | Offline behaviour |
|---|---|---|
| Static assets (JS, CSS, HTML) | Pre-cache | Always available |
| Google Fonts | Cache First | Cached after first load |
| Supabase REST API | Network First (5s timeout) | Falls back to cached data |
| Supabase Auth | Network Only | Fails gracefully |
| AI Edge Functions | Network Only | Shows "offline" message |
| Page navigations | Network First (3s timeout) | Serves cached index.html |
| Images | Stale While Revalidate | Shows cached, updates in background |

## Service worker update flow

With `registerType: "autoUpdate"`:
1. New deploy → Vite generates new SW with updated file hashes
2. Browser detects changed SW on next page load
3. SW installs in background silently
4. On next page navigation, new SW activates (`skipWaiting: true`)
5. User sees updated app without any manual action needed
